02传统痛点PAIN

设计师产出静态视觉稿速度慢，且无法在真实环境中验证交互逻辑与动态效果。

03能做什么WHAT IT DOES

01 · 规范 内置统一设计规范

所有页面共用同一套按钮、表单、卡片、图标等组件，AI 生成的内容自动遵循团队视觉标准，保持风格一致。

02 · 推进 分角色分步骤推进

需求理解、原型设计、组件搭建、页面装配，每个环节由专门的 AI 模块独立处理，各环节职责清晰，避免单一模型一次性生成带来的质量波动。

03 · 流程 走完完整的设计流程

从输入一句业务需求开始，经过结构化的多步处理，最终直接输出可在浏览器中打开、可点击交互的 HTML 页面。

04 · 优化 支持持续升级优化

将全流程中产生的问题、异常记录，这些数据支持系统进行迭代优化，使后续生成质量不断提升。

02AI 设计团队AGENTS · 6

REQUIREMENT-AGENT

需求采集员

核心职责是与需求方直接沟通，按照标准模板将"要做什么"问清楚、问透彻，最终输出一份结构完整、无歧义的需求文档。

COMPONENT-AGENT

零件维护员

负责 components/ 目录下所有设计组件的开发、维护与迭代，按业务需要新增组件，或扩展现有组件的功能与状态。

CSS-AGENT

样式标准员

负责 css/ 目录下全局样式标准的制定与维护，包括颜色体系、字号梯度、间距尺度、圆角、阴影等设计令牌的统一管理。

FONT-AGENT

图标管理员

负责 fonts/ 目录下图标库的维护与管理，所有新图标需先经过归一化处理后方可入库，确保风格统一、调用规范。

FRAGMENT-AGENT

骨架架构师

负责 fragment/ 目录下页面整体框架的设计与维护，定义侧边栏、顶部栏、内容区等布局规则，以及抽屉、弹窗等叠加层的弹出逻辑与位置规范。

PAGE-AGENT

页面装配员

拿到确认后的需求文档，从组件库中选取所需零件，按照页面骨架的布局规则，将各个组件拼装成最终可交付的页面。

03执行流程EXECUTION FLOW

适合一张表单、一个列表这类独立功能，比如设计一个员工设备申请单。

01

明确需求

问清为什么做、给谁用、做什么、展示什么，以及空数据、加载中、出错等边界情况如何处理。

02

画原型

先输出一张 ASCII 草图，标明标题区、表单区、按钮区的位置。确认结构无误后再继续。

03

页面拼装

草图确认后，AI 从组件库选取组件、调用样式变量、遵循框架骨架，拼装成最终的 HTML 页面。

04

自检与交付

页面先完成自我检查。交付一个文件夹，里面是完整代码，浏览器直接可打开运行。

适合"一整套系统"的场景，比如会议室预订后台，涉及查找会议室、查看详情、我的预订、通知中心等多个页面。

01

明确需求

确定给谁用、拆分边界、一级导航项数、各内容的承载形态（独立页/抽屉/弹窗），产出页面清单总表。

02

画原型

先绘制所有页面共用的骨架（侧边栏+顶部栏），确认后再逐张绘制各页面内容区。

03

页面拼装

先搭空壳外框确认布局，再一次性定死所有共用规范，最后并行装配全部页面内容区。

04

交付 + 自动检查

所有页面通过菜单链接串联打通，自动扫描重复 ID、失效链接、违反统一标准的情况，发现问题即回修。

04复盘机制FEEDBACK LOOP

FEEDBACK LOOP
每一次踩坑，
都变成下一次的规则。

项目内设有一份专门的复盘文档，用于记录所有 AI 与主会话中踩过的坑。每条记录固定包含三段：现象、后果、教训。每次 AI 跑偏，便新增一条记录；同类问题反复出现时，则将对应规则固化进对话模板或 Agent 描述中，从源头杜绝再犯。

MODULE · 01 / 07
导向
DASHBOARDHOMEPAGE

日导向：让每一天都从清晰的目标开始。日导向帮助每个人在一天伊始，就明确今日的核心工作任务，聚焦要事。为适应不同岗位的个性化需求，它还支持自定义字段，员工可按需补充关键信息，让每日规划更贴合实际工作场景。

周月导向：将周期性的成果汇报，沉淀为可延续的过程资产。导向模块将这一流程完整线上化，一键发布、在线填写、到点开会，无需人工收集协调。汇报中涉及的任务与协同事项，可直接创建为任务和会议，可持续追踪、随时回溯，让每一次填写都转化为可延续的过程资产。

MODULE · 02 / 07
消息
IMNOTIFY

消息系统重新定义了"消息"的角色——它不再只是聊天工具，而是信息流动的入口，协作发生的地方，是业务状态的实时载体，也是所有系统连接的通道。通过高性能实时通信引擎、可扩展的消息架构与智能化数据处理能力，消息系统将分散在各个系统中的通知、记录与操作整合到同一个入口。它让信息自然沉淀，让协作就地完成，让业务在对话中直接流转，用户无需再在系统间来回辗转。

MODULE · 03 / 07
任务
TODOASSIGN

传统的项目管理工具，大多只解决了把任务派下去这一件事。但任务一旦派出去，围绕它的讨论、文件、进度往往就散落在各处，事情做完了，过程却没留下。为此，我们让每一个任务都承载一件事的完整生命周期。任务不仅包含标题，还内嵌负责人、参与人、起止时间、优先级、工时与描述等要素，大任务可层层拆解为子任务，将责任落到每个人头上。围绕这件事的沟通、文件与动态全部内嵌在任务详情里，事情做完，完整的过程也留在了任务里。同时支持列表、看板、甘特图、工时等多视图自由切换，可将常用筛选存为个人视图，真正做到"一份数据、千人千面"。

MODULE · 04 / 07
会议
MEETINGMINUTES

不止于音视频连接，更是从发散到聚焦的共识达成闭环。传统的线上会议工具往往只解决了"能开会"的问题，但一场高效会议的难点在于会前准备、会中聚焦和会后落实。我们的会议模块打造了完整闭环：会前无需人工收集，主讲人可直接在线分享PPT汇报；会中支持参会者匿名填写观点、实时投票，实现从思维发散到聚焦收敛的高效决策流程；会后产生的决策与待办事项可一键转为任务或关联至会议纪要，让结论不止停留在屏幕里，而是立即转化为可追踪的行动。

MODULE · 05 / 07
文档
DOCSCO-EDIT

让文档成为协同的起点，从知识到行动无缝衔接。过去的文档工具本质上是"网盘+编辑器"，文档写完就静静躺在文件夹里，知识沉淀下来却很难变成行动。为此，我们让文档成为协同的起点：文档里读到的任何待办事项，选中即可一键创建任务，指定负责人、设好截止日期，直接落到他的「我的任务」里；需要拍板的议题，可一键发起会议，拉上参会人，并将该文档设为会议议程，会后的纪要又能回写到同一篇文档。读着读着，就把活派了出去、把会开了起来，无需在系统间来回切换。

MODULE · 06 / 07
表单
FORMLOW-CODE

一切"数据驱动的结构化界面"，皆可由表单引擎统一构建。只要背后有数据模型和视图骨架，它就是表单引擎的统治领域，重新定义了信息收集、展示与交互的方式。从标准业务流转如请假单、日导向，到纯粹的数据详情展示如会议详情页、审批只读页，再到活动大促页、复杂逻辑的问卷调查等富视觉营销交互，都可以通过表单引擎灵活搭建。更进一步，表单引擎能让静态内容"活化"：可在文档中无缝插入投票、抽奖、评分等微表单组件，甚至嵌入完整业务表单，让文档从单向信息变为双向互动。

MODULE · 07 / 07
审批
APPROVALFLOW

让每一次审批都推动业务前进，而非止步于签字。传统的审批往往是被动的"同意"或"驳回"，审批完了流程就结束了，与业务执行是割裂的两张皮。为此，我们将审批流深度融入业务，从提交审批的那一刻起，便与表单引擎无缝结合，无论是标准业务表单还是自定义数据，都能直接发起流转。审批不仅仅是意见的填写，其结果能直接触发下一阶段的业务动作：自动生成待办任务、更新关联文档状态、或在审批通过后拉启一个新项目。整个过程透明可追溯，每一次决策都沉淀为业务数据，让审批从一个孤立的签字环节，转变为驱动业务自动流转的关键引擎。
