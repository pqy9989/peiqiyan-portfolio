Q1 · 角色 + 高频场景

决定默认落地页 · 首屏重心 · 是否角色变体

服务角色 SINGLE-ROLE

  • 仅员工（单一角色）
  • 主角色 + 次角色 · 同套页面按差异展示
  • 多角色合并视角

角色清单

主角色
普通员工（含代他人预约 —— PDF UC5.2 明确说代订只需操作者的普通预订权限）
心智状态
日常办公场景中临时穿插的"订房子事"，希望快速搞定
使用频次
每月 2~10 次

最高频 2~3 个场景 MULTI-CHOICE

  • 提前排会 —— 提前几天/几周订周会、季度会、外部会谈（依赖日历选择器 + 参会人选择器 + 周期预约）
  • 查自己的会 —— 看今天/本周要去哪间会议室、几点开始、和谁开
  • 当下救急订房
  • 临时变更（改时间/换房/取消/提前结束）
Q2 · UC 合并/拆分策略

决定页数减法

能力重叠 UC

  • UC3.7 找会议室列表 + UC5.8 占用时间轴 → 合一页 + 顶部 view 切换（"找会议室"）
  • UC5.1 查询可用会议室 不单独成页 —— 是"找会议室"页的筛选行为
  • 单独成页：UC3.8 会议室详情 · UC5.9 预约详情 · UC5.7 我的预约 · UC10 通知中心

主 + 衍生 TOGGLE

  • 同一表单 + toggle 展开
  • 单次预约 / 周期预约 在同一抽屉，"重复" toggle 展开周期规则
  • 创建 / 编辑预约 共用同一抽屉，按是否带预约 ID 决定模式
  • 落位：booking-form 抽屉
Q3 · 一级导航 + 默认落地页

决定页面骨架

一级导航项 SIDEBAR · 2 入口

  • 入口 1：📅 我的预约
  • 入口 2：🔍 找会议室
  • 不进 sidebar：通知（顶栏右上 🔔）、管理后台（仅有权限时显示在 sidebar 底部）

默认落地页

  • 📅 我的预约 ← 高频"查自己的会"配对推荐
Q4 · 表单 + 详情 + 轻量功能 的承载形态

决定哪些是独立页面 · 页数最终定下来

主要创建/编辑表单

  • 右侧抽屉 —— booking-form，叠加在任何页之上

业务对象详情

  • 跳独立页 —— room-detail、booking-detail 都是独立 routed 页
  • 理由：动作多（编辑/取消/提前结束/开线上）、内容多、需要分享链接

轻量常用功能（通知）

  • 右上铃铛 + 下拉面板 + 「查看全部」跳独立通知中心页

PROTO-2 · 内容区头部 content-header

两种变体

变体 A · 列表型
适用：我的预约 / 找会议室 / 通知中心
┌─ content-header ─────────────────────────────────────────────┐
│  一级标题                                          [搜索 🔍] │
│  <副标题占位>                                                 │
└──────────────────────────────────────────────────────────────┘
变体 B · 详情型
适用：会议室详情 / 预约详情
┌─ content-header ─────────────────────────────────────────────┐
│  [← 返回]  一级标题                       · 状态徽标         │
│            <副标题占位：关键摘要（时间、位置等）>             │
└──────────────────────────────────────────────────────────────┘
  • 右上角禁止画主 CTA（已被 sidebar 接住）
  • 右上角只允许放：搜索 / 视图切换 / 次要操作（导出 / 批量 / 刷新）
  • 详情型统一带 [← 返回]，标题后跟状态徽标
