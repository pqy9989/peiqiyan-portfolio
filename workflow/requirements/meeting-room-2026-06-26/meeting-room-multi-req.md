## WHY · 这组页面为什么存在
- **价值**：让企业员工在桌面端独立完成会议室预约 —— 提前排周会/季度会，随时查自己今天/本周要去哪间会议室
- **不做会怎样**：员工只能在 IM 群口头协商或用 Excel 表抢房，冲突频发、责任难追、无法批量管理
- **覆盖范围**：
  - **覆盖**：UC1.5~7 普通预订策略（仅前端校验展示）、UC2 层级（前端读取层级名/位置用）、UC3.7~8 用户端会议室列表/详情、UC5 预约 CRUD/查询/详情、UC6.1~2 线上会议复用、UC8.1~3 会议开始/提前结束/正常结束、UC9 周期预约、UC10 通知
  - **不覆盖**：
    - 去管理端：UC1.1~4 管理权限、UC1.8~10 预订管理员、UC2.2~4 层级 CRUD、UC3.1~6/9~10 会议室管理、UC4 配置中心、UC11 任务、UC12 统计、UC13 审计
    - 已删除：UC7 KR 全部、UC5/UC6 中 KR 相关子流程
    - 后续规划：UC4.4 签到、UC4.6 主题可见配置、UC8.4 抢占、位置变化通知

## Q1 · 角色 + 高频场景
> 决定默认落地页、首屏重心、是否要"角色变体"区块

- **服务角色**：
  - ☑ 单一角色（员工）
  - ⬜ 主角色 + 次角色
  - ⬜ 多角色合并视角

- **角色清单**：
  - 主角色：**普通员工**（包含代他人预约能力 —— PDF UC5.2 明确说代订只需操作者的普通预订权限）
  - 心智状态：日常办公场景中临时穿插的"订房子事"，希望快速搞定
  - 使用频次：每月 2~10 次

- **最高频 2~3 个场景**：
  1. **提前排会** —— 提前几天/几周订周会、季度会、外部会谈（依赖日历选择器 + 参会人选择器 + 周期预约）
  2. **查自己的会** —— 看今天/本周要去哪间会议室、几点开始、和谁开

## Q4 · 表单 + 详情 + 轻量功能 的承载形态
> 决定哪些是独立页面、页数最终定下来

- **主要创建/编辑表单**：
  - ☑ 右侧抽屉 —— `booking-form`，叠加在任何页之上

- **业务对象详情**：
  - ☑ 跳独立页 —— `room-detail`、`booking-detail` 都是独立 routed 页

- **轻量常用功能**（通知）：

## PROTO · 原型输出
> Q1~Q4 + PAGES 确定后产出。先画统一框架（sidebar + content-header），再画每一页完整原型。
>
> ⚠️ **PROTO 只是设计沟通用的参考图，不是实现规格**。
> page-agent 装配时**直接引用** `fragment/fragment.html` 作为页面骨架，不照着 ASCII 重写 sidebar / content-header 的结构。
> 原型用于对齐 "菜单项填什么 / 字段怎么放 / 哪些区段出现"，**容器层面由 fragment 接管**：
> - sidebar 的宽度、收起按钮、底部信息位 → fragment 提供，本段只填菜单项内容
> - content-header 的高度、边距、左右槽位 → fragment 提供，本段只决定填哪个变体（列表型 / 详情型）
> - main-area 的滚动行为、自适应 → fragment 提供，本段只填字段落位

### PROTO-1 · 二级导航 sidebar（统一框架，所有页共用）

```
┌─ 二级导航 sidebar ──────────┐
│  会议室          [收起 ⇋]   │
│  <副标题占位>                │
│                            │
│  ┌──────────────────────┐  │
│  │     [+ 新建预约]      │  │  ← 主 CTA，固定 sidebar 顶部
│  └──────────────────────┘  │
│                            │
│  📅  我的预约     ● 默认选中 │
│  🔍  找会议室               │
│                            │
│ ─────────────────────────  │
│  ⚙   管理后台   (有权限时)  │
└────────────────────────────┘
```

**规则**：
- 主 CTA `[+ 新建预约]` 唯一，固定 sidebar 顶部（不放 content-header）
- 菜单项只 2 个（我的预约 / 找会议室），不需要三级
- 通知不进 sidebar（走顶栏铃铛）
- 「管理后台」仅当登录人有会议室管理权限时显示

### PROTO-2 · 内容区头部 content-header（两种变体）

#### 变体 A · 列表型
适用：我的预约 / 找会议室 / 通知中心

```
┌─ content-header ─────────────────────────────────────────────┐
│  一级标题                                          [搜索 🔍] │
│  <副标题占位>                                                 │
└──────────────────────────────────────────────────────────────┘
```

#### 变体 B · 详情型
适用：会议室详情 / 预约详情

```
┌─ content-header ─────────────────────────────────────────────┐
│  [← 返回]  一级标题                       · 状态徽标         │
│            <副标题占位：关键摘要（时间、位置等）>             │
└──────────────────────────────────────────────────────────────┘
```

**规则**：
- 右上角**禁止画主 CTA**（已被 sidebar 接住）
- 右上角只允许放：搜索 / 视图切换 / 次要操作（导出 / 批量 / 刷新）
- 详情型统一带 `[← 返回]`，标题后跟状态徽标

## EDGE · 项目级约束

- **统一交互规范**（写在这里，单页 brief 不重复）：
  - 越权按钮：主动操作 → 灰 + tooltip 显示原因；身份无关 → hidden；路由级 → 跳无权页
  - 危险操作：弹窗二次确认（取消预约 / 提前结束会议）
  - 成功反馈：toast 右上；失败反馈：表单 / 列表行内 inline
  - 通知触达：仅站内通知中心（推送 / 邮件由后端决定，UI 不画渠道开关）
  - 审计：预约详情展示简版状态时间线（创建 / 修改 / 取消 / 提前结束 / 释放），其他日志走管理端 UC13

- **统一字段约定**：
  - 预约状态徽标颜色：
    - CONFIRMED → 蓝
    - IN_PROGRESS → 绿
    - COMPLETED → 灰
    - CANCELLED → 灰 + 删除线
    - RELEASED → 黄
  - 会议室状态：ENABLED / DISABLED（临时禁用 / 永久禁用 二级状态）
  - 时间显示：企业时区，统一 `YYYY-MM-DD HH:mm` 或相对时间（"今天 14:00"）
  - 通知未读样式：左侧蓝点
  - 设备字典：7 类预设（投影 / 电视大屏、白板、视频会议终端、麦克风 / 扬声器、摄像头、电话、HDMI / 转接器），筛选用 chip，卡片用图标
  - 会议室"名+位置"：单行展示，统一在 list / card / detail 中规整

- **私有 CSS 前缀**：`--meeting-room-*`

- **视口**：仅桌面 1440+

- **空态文案**：
  - 我的预约空：插画 + "你还没有任何会议" + `[+ 新建预约]` CTA
  - 找会议室无匹配：插画 + "没找到匹配的会议室" + `[清空筛选]`
  - 通知中心空：插画 + "暂无通知"
