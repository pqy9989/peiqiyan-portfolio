01 · WHY

为什么存在

价值
新员工入职 3 天内 1 分钟内提交设备申请，IT 备货发货，到岗即可领齐装备。
不做会怎样
邮件 / Slack / 口头流转，IT 无法预判库存，新员工到岗设备没准备好，影响入职体验。
上下游
上游 = 入职流程（HR 通知 / 排班） · 下游 = 审批中心 → IT 备货 / 发货 → 物流跟踪
02 · WHO

给谁用

主角色
新入职员工 · 低频赶时间（年度 1 次，入职第 1 周内填，希望几分钟完成）
次角色
—（部门长 / IT 审批由审批中心承担；老员工补领由本页兼容）
03 · DO

要做什么 + 怎么做

唯一主任务
从入职套餐快选设备并提交申请
唯一主 CTA
+ 新建申请 放在 navbar.slot-action（与卡片"一键领取"通往同一抽屉）
交互形态
右侧抽屉填表（不分步向导——赶时间用户优先少点击）

最短步骤（4 步）

  1. 进入页面（首屏看到套餐 + 列表 / 工作台）
  2. 点击某个匹配岗位的套餐 / navbar + 新建申请
  3. 右侧抽屉打开，补 3 个字段：交付地址、期望到货日、备注
  4. 点"提交" → toast + 抽屉关闭 + 高亮新项

危险操作

  • 提交后撤回需二次确认（仅"待审批"可撤回）
  • 删除草稿需二次确认
  • 抽屉填到一半关闭时弹"保存草稿 / 放弃"
04 · SHOW

呈现什么 + 怎么组织

布局骨架
套餐展示区 + 右抽屉（具体卡片墙还是工作台见 PROTO 段）
数据量级
每位员工 1-2 张 / 年，无需分页 / 虚拟滚动
默认排序
创建时间 desc

F1 字段（必看，5 个）

字段数据源显示形式
套餐识别 套餐.name + 套餐.suitablePositions 卡片主标题
设备清单 + 库存状态 套餐.items[] / 套餐.stockStatus "笔记本 + 显示器 + 鼠键 · 现货可领"
预估总价 + 期望到货日 套餐.price / 申请.expectedDate "¥ 18,500 · 2026-07-05"，tabular-nums
申请状态徽标 + 审批进度 申请.status / 申请.approvalProgress 草稿 / 待审批 / 已批准 / 发货中 / 已领取 / 已驳回 / 已取消（cs-status-badge）
物流状态 + 预计到货日 申请.shippingStatus / 申请.eta "京东物流 · 预计 06-30 送达"

F2 字段（折叠 / 抽屉详情）

审批人 / 创建时间 / 修改记录 / 联系电话 / 工号

F3 字段（hover tooltip / 详情页）

申请单 ID / SKU 编码 / 套餐内部编码

菜单结构（navbar 二级导航 4 项）

  1. 套餐选择（data-active 入口）
  2. 我的申请
  3. 设备目录（浏览所有设备，不能从此页直接申请）
  4. 草稿箱

分组维度

  • OPTION-1 cs-filter-bar tab 切状态（全部 / 草稿 / 待审批 / 已批准 / 发货中 / 已领取 / 已驳回）
  • OPTION-2 工作台 3 卡块按生命周期切片（推荐 / 进行中 / 物流跟踪），无 filter-bar

06 · EDGE

边界 + 约束 + 成功

5 种状态

EMPTY
插画 + 引导文案 + 大号主 CTA 高亮在 navbar.slot-action（option-2 时空态体现在卡块 A 推荐区）
LOADING
套餐卡片骨架屏（4 卡占位）；抽屉字段 cs-skeleton
ERROR
网络异常 → 显示重试按钮 + 本地缓存（若可用）；字段校验失败 → 红色提示在表单
SUCCESS
toast + 抽屉关闭 + 留在当前页 + 高亮新项（淡蓝底 2s 渐消）
NO PERM
HR 入职登记未完成 → 引导联系 HR

私有 CSS 前缀
