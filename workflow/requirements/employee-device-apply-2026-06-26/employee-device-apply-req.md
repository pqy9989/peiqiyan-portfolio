## WHY · 为什么存在
- **价值**：新员工入职 3 天内 1 分钟内提交设备申请，IT 备货发货，到岗即可领齐装备
- **不做会怎样**：邮件 / Slack / 口头流转，IT 无法预判库存，新员工到岗设备没准备好，影响入职体验
- **上下游**：上游=入职流程（HR 通知/排班）；下游=审批中心 → IT 备货/发货 → 物流跟踪

## WHO · 给谁用
- **主角色**：新入职员工 · 低频赶时间（年度 1 次，入职第 1 周内填，希望几分钟完成）
- **次角色**：—（部门长 / IT 审批由审批中心承担；老员工补领由本页兼容）

## DO · 要做什么 + 怎么做
- **唯一主任务**：从入职套餐快选设备并提交申请
- **最短步骤**（4 步）：
  1. 进入页面（首屏看到套餐 + 列表/工作台）
  2. 点击某个匹配岗位的套餐 / navbar `+ 新建申请`
  3. 右侧抽屉打开，补 3 个字段：交付地址、期望到货日、备注
  4. 点"提交" → toast + 抽屉关闭 + 高亮新项
- **唯一主 CTA**：`+ 新建申请` 放在 navbar.slot-action（与卡片"一键领取"通往同一抽屉）
- **交互形态**：右侧抽屉填表（不分步向导——赶时间用户优先少点击）
- **危险操作**：
  - 提交后撤回需二次确认（仅"待审批"可撤回）
  - 删除草稿需二次确认
  - 抽屉填到一半关闭时弹"保存草稿 / 放弃"

## SHOW · 呈现什么 + 怎么组织
- **布局骨架**：套餐展示区 + 右抽屉（具体卡片墙还是工作台见 PROTO 段）
- **F1 字段**（必看，5 个）：
  1. **套餐识别** — `套餐.name + 套餐.suitablePositions` — 卡片主标题
  2. **设备清单 + 库存状态** — `套餐.items[] / 套餐.stockStatus` — "笔记本 + 显示器 + 鼠键 · 现货可领"
  3. **预估总价 + 期望到货日** — `套餐.price / 申请.expectedDate` — "¥ 18,500 · 2026-07-05"，tabular-nums
  4. **申请状态徽标 + 审批进度** — `申请.status / 申请.approvalProgress` — 草稿 / 待审批 / 已批准 / 发货中 / 已领取 / 已驳回 / 已取消（用 cs-status-badge）
  5. **物流状态 + 预计到货日**（option-2 独有）— `申请.shippingStatus / 申请.eta` — "京东物流 · 预计 06-30 送达"
- **F2 字段**（折叠 / 抽屉详情）：审批人 / 创建时间 / 修改记录 / 联系电话 / 工号
- **F3 字段**（hover tooltip / 详情页）：申请单 ID / SKU 编码 / 套餐内部编码
- **数据量级**：每位员工 1-2 张/年，无需分页 / 虚拟滚动
- **菜单结构**（navbar 二级导航 4 项）：
  1. 套餐选择（`data-active` 入口）
  2. 我的申请
  3. 设备目录（浏览所有设备，不能从此页直接申请）
  4. 草稿箱
- **分组维度**：
  - option-1：`cs-filter-bar` tab 切状态（全部 / 草稿 / 待审批 / 已批准 / 发货中 / 已领取 / 已驳回）
  - option-2：工作台 3 卡块按生命周期切片（推荐 / 进行中 / 物流跟踪），无 filter-bar
  - 默认排序：创建时间 desc

## PROTO · 内容区域布局原型图
- **方案数量**：☑ 多选 2 个并行（结构层面差异）
- **表达载体**：ASCII 框图（直接内嵌本段，不另建 proto/ 子目录）
- **多选下游派单**：**并行**派 2 个 page-agent，各装配 `pages/employee-device-apply-option-1/` 和 `pages/employee-device-apply-option-2/`，验收方案需在结构层面有差异，**不能**只换配色字号

### option-1 · 套餐卡片墙 + filter-bar + 右抽屉

```
┌─ content-header ────────────────────────────────────────┐
│ 标题：<占位>   ·   副标题：<占位>                          │
├─ filter-bar（按申请状态切 tab）──────────────────────────┤
│ [全部][草稿][待审批][已批准][发货中][已领取][已驳回]        │
├─ 套餐卡片墙（3-4 列）─────────────────────────────────────┤
│ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐                     │
│ │ F1·套餐识别                                             │
│ │ F1·设备清单 + 库存徽标                                  │
│ │ F1·预估总价 + 期望到货日                                │
│ │ [次 CTA·领取]                                           │
│ └──────┘ └──────┘ └──────┘ └──────┘                     │
│ ▾ 我的申请（折叠区；F1·状态徽标 + 审批进度）              │
└──────────────────────────────────────────────────────────┘
       └─ 抽屉（点卡片 / navbar 主 CTA 打开同一个）─────────┐
          │ 套餐预览 / 交付地址 / 期望到货日 / 备注          │
          │ [次 CTA·保存草稿]            [次 CTA·提交]      │
          └────────────────────────────────────────────────┘
```

**设计意图**：
- 扁平卡片墙 + filter-bar 状态 tab 切换：用户先按状态过滤，再在剩余结果里挑套餐
- 卡片本身是主信息密度区，状态过滤靠 tab
- "我的申请" 默认折叠，避免和套餐墙竞争注意力

**F1 字段落位**：

| F1 字段 | 位置 |
|---|---|
| 套餐识别 | 卡片主标题 |
| 设备清单 + 库存徽标 | 卡片正文 |
| 预估总价 + 期望到货日 | 卡片底部价格区 |
| 申请状态徽标 + 审批进度 | "我的申请"折叠展开区 |

### option-2 · 工作台 3 卡块 + 右抽屉（无 filter-bar）

```
┌─ content-header ────────────────────────────────────────┐
│ 标题：<占位>   ·   副标题：<占位>                          │
├─ 工作台 3 卡块（按申请生命周期切片，替代状态 tab 切换）──┤
│ ┌─ A·推荐角色 ─┐ ┌─ B·进行中角色 ─┐ ┌─ C·物流跟踪角色 ─┐ │
│ │F1·套餐识别  │ │F1·套餐识别      │ │F1·套餐识别       │ │

│ │F1·总价+到货 │ │  审批进度       │ │  预计到货日       │ │
│ │[次 CTA·领取]│ │  进度条 n/N     │ │                  │ │
│ │             │ │[次 CTA·撤回]    │ │[次 CTA·物流详情] │ │
│ │             │ │[次 CTA·查看]    │ │[次 CTA·联系 IT]  │ │
│ └─────────────┘ └─────────────────┘ └──────────────────┘ │
└──────────────────────────────────────────────────────────┘
       └─ 抽屉（点 A 次 CTA / navbar 主 CTA 打开）───────────┐
          │ 套餐预览 / 交付地址 / 期望到货日 / 备注          │
          │ [次 CTA·保存草稿]            [次 CTA·提交]      │
          └────────────────────────────────────────────────┘
```

**设计意图**（与 option-1 的结构差异）：
- 按生命周期切片：3 卡块本身就是状态分组（推荐 / 进行中 / 物流跟踪），**无需** filter-bar
- 同屏看到所有阶段：避免 tab 切换的视线跳动
- F1 字段分散到 3 卡块：每个卡块承担一类信息密度

**F1 字段落位**：

| F1 字段 | 位置 |
|---|---|
| 套餐识别 | 卡块 A 主体 + 卡块 B/C 顶部识别行 |
| 设备清单 + 库存 | 卡块 A |
| 预估总价 + 期望到货日 | 卡块 A 底部价格区 |
| 申请状态徽标 + 审批进度 | 卡块 B（含进度条 step n/N） |
| 物流状态 + 预计到货日 | 卡块 C |

## EDGE · 边界 + 约束 + 成功
- **5 种状态**：
  - **空态**：插画 + 引导文案 + 大号主 CTA 高亮在 navbar.slot-action（option-2 时空态体现在卡块 A 推荐区）
  - **加载态**：套餐卡片骨架屏（4 卡占位）；抽屉字段 `cs-skeleton`
  - **错误态**：网络异常 → 显示重试按钮 + 本地缓存（若可用）；字段校验失败 → 红色提示在表单
  - **成功态**：toast + 抽屉关闭 + **留在当前页** + 高亮新项（淡蓝底 2s 渐消）
  - **无权限态**：HR 入职登记未完成 → 引导联系 HR
- **私有 CSS 前缀**：`--eda-*` / `.eda-*`（employee-device-apply）
- **视口**：仅桌面 1440+（6 列卷轴）
- **成功指标**：
  - 80% 新员工入职 3 天内提交设备申请
  - 从进页到 toast 弹出 < 90 秒（option-1 卡片墙）/ < 30 秒（option-2 一键领取场景）
